> A lay-of-the-land for the directories under `~/.claude/` not covered in earlier sections. Enough to know what each one does and whether you should ever touch it.

---

## Keyboard shortcuts

`~/.claude/keybindings.json` does not exist until you create it. It maps key combinations to Claude Code TUI actions. Run `/keybindings-help` to see the format, or create it manually.

---

## Templates

`~/.claude/templates/CLAUDE.md` is a starter scaffold for new project `CLAUDE.md` files. It is not auto-applied — you copy it into a new project's root and fill in the blanks. Useful if you set up projects often.

---

## Plugin state

Plugins write their runtime state under `~/.claude/channels/` (communication integrations), `~/.claude/docs/` (cached documentation), and `~/.claude/plans/` (plan files from the planning skill).

- **`channels/`** — integration state for communication plugins (bot PIDs, access maps, approved threads). Safe to delete when reinstalling a plugin.
- **`docs/`** — local documentation cache fetched by plugins or context MCPs. Safe to clear stale subdirectories.
- **`plans/`** — Markdown plan files auto-generated by the planning skill. Accumulate indefinitely. Prune completed plans periodically.

---

## Session and task tracking

Three directories managed by Claude Code core:

- **`sessions/`** — one JSON file per running Claude Code process, keyed by PID. Tracks session ID, working directory, start time, and entrypoint. Auto-purged when processes exit.
- **`tasks/`** — state files for dispatched agent tasks (SDK managed-agent infrastructure). One file per parallel agent run.
- **`todos/`** — in-session TODO lists, one file per session UUID. Cleared on session end or `/compact`. JSON arrays with `id`, `content`, `status`, and `priority` fields.

---

## Shell snapshots

`~/.claude/shell-snapshots/` holds one snapshot file per session. Each captures the full shell environment (env vars, PATH, shell functions) before a Claude Code session starts, so subshells spawned during tool execution get a clean, reproducible environment.

One file per session. Safe to prune old ones manually — keep a few recent ones in case you need to debug a tool execution issue.

---

## Paste cache and history

- **`paste-cache/`** — per-session files storing slash-command and paste events. A `latest` symlink points to the most recent session. Grows unboundedly; safe to archive or delete old session files.
- **`history.jsonl`** — the complete TUI command history across all sessions. Each line is a JSON object with the display text, timestamp, project path, and session ID. No automatic rotation. Safe to truncate if it gets large.

---

## Telemetry

`~/.claude/telemetry/` stores events that failed to upload to Anthropic's telemetry pipeline (network errors, etc.). Anthropic retries on the next run; files auto-purge after a successful retry and only linger on persistent failures.

Treat this directory as sensitive. Do not share its contents or enumerate what fields it contains — it holds session metadata that is personal to your installation.

---

## Statsig

`~/.claude/statsig/` is written by the Statsig SDK embedded in Claude Code. It handles feature flags and A/B testing. The directory contains a persistent device identifier, a session identifier, and a queue of failed log events.

The device identifier persists across reinstalls until you delete it manually. It is pseudonymous (not directly tied to your account), but treat it as sensitive if you share dotfiles publicly.

---

## Usage data and stats cache

- **`usage-data/`** — an HTML dashboard (`report.html`) plus JSON files backing it. Generated periodically. Safe to delete freely — regenerated on next run.
- **`stats-cache.json`** — a pre-computed snapshot used to render the `/usage` dashboard without re-scanning history on every open. Recomputed incrementally when sessions close.
- **`cache/`** — a cached copy of the plugin system changelog. Single file, low overhead.

---

## IDE and debug

- **`ide/`** — stores IDE integration state (connection info, port bindings, workspace mappings) when a Claude Code IDE extension is active. Empty when no extension is connected.
- **`debug/`** — plain-text stream logs from the IDE integration layer. Useful when diagnosing stuck subagent pipelines or IDE transport failures. Auto-rotated.
- **`logs/`** — reserved for future structured log output. Currently unused.

---

## File history

`~/.claude/file-history/` stores pre-edit snapshots of every file Claude has modified. Each edited file path gets its own UUID-named subdirectory. Versions accumulate (`@v1`, `@v2`, ...) and there is no automatic pruning.

This is the directory most likely to grow without you noticing. Clean it up quarterly.

---

## Plugin registry files

Three files under `~/.claude/plugins/`:

- **`blocklist.json`** — plugins blocked from installation. Includes test entries from Anthropic's safety pipeline. You can add your own entries here to block specific plugins system-wide.
- **`known_marketplaces.json`** — maps marketplace IDs to their metadata. Updated on `claude plugins update`.
- **`install-counts-cache.json`** — cached popularity data from the marketplace API, used to rank plugins in the browser. Refreshed periodically.

---

## Miscellaneous

- **`backups/`** — reserved for automatic backups of `settings.json` before destructive changes. Currently unused in most setups.
- **`session-handoffs/`** — intended for cross-session handoff notes written at session end. Empty by default; some setups use an external vault instead.
- **`.caveman-active`** — a flag file written by certain plugins to signal a minimal-output mode. Delete it to disable the mode, or trace the hook that writes it.

---

## Retention at a glance

| Path | Notes |
|---|---|
| `~/.claude/file-history/` | Grows indefinitely — prune quarterly |
| `~/.claude/paste-cache/` | Archive old session files periodically |
| `~/.claude/history.jsonl` | No rotation — truncate when large |
| `~/.claude/usage-data/` | Delete freely, regenerates |
| `~/.claude/plans/` | Delete completed plans |
| `~/.claude/shell-snapshots/` | Keep last few, prune the rest |
| `~/.claude/telemetry/` | Treat as sensitive, auto-clears |
| `~/.claude/statsig/` | Sensitive — do not share publicly |
