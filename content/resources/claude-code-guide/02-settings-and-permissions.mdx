> `settings.json` is where Claude Code gets its boundaries. What tools it can use, which commands run without asking, what's off-limits entirely. Done right, you get autonomous work on routine stuff and a confirmation prompt before anything destructive.

---

## 1. Top-Level Behavior Settings

| Key | Value | What it does |
|-----|-------|--------------|
| `defaultMode` | `"bypassPermissions"` | Claude acts without a permission prompt on every tool call. The `allow`/`ask` lists still apply — this just removes the generic runtime confirmation. |
| `skipDangerousModePermissionPrompt` | `true` | Suppresses the one-time startup warning when `bypassPermissions` is active. Sessions start silently. |
| `advisorModel` | `"opus"` | Model used when `advisor()` is called for deep-reasoning review. Set to Claude Opus. |
| `autoUpdatesChannel` | `"latest"` | Auto-updates from the stable `latest` channel. |
| `showThinkingSummaries` | `true` | Renders extended thinking summaries in the UI. |

---

## 2. Environment Variables

Defined in `settings.json` → `env`:

```json
"env": {
  "CLAUDE_CODE_DISABLE_AUTO_MEMORY": "1",
  "CLAUDE_CODE_SUBAGENT_MODEL": "sonnet"
}
```

| Variable | Value | Purpose |
|----------|-------|---------|
| `CLAUDE_CODE_DISABLE_AUTO_MEMORY` | `"1"` | Disables built-in automatic memory/summarization. Memory is managed manually via `claude-mem`, the knowledge graph, or vault notes. |
| `CLAUDE_CODE_SUBAGENT_MODEL` | `"sonnet"` | Sets the default model for spawned subagents to Sonnet. |

## 4. Permissions — `ask` Array (Destructive Operations)

These require explicit confirmation before execution:

```text
Bash(rm *)               — file deletion
Bash(rmdir *)            — directory removal
Bash(shred *)            — secure file deletion
Bash(git push --force*)  — force push (history rewrite)
Bash(git reset --hard*)  — hard reset (discards local changes)
Bash(git clean -f*)      — remove untracked files
Bash(git clean -fd*)     — remove untracked files + dirs
Bash(git checkout -- *)  — discard file changes
Bash(git checkout .*)    — discard all working-tree changes
Bash(git restore *)      — restore files (discard changes)
Bash(DROP TABLE*)        — SQL table deletion
Bash(DROP DATABASE*)     — SQL database deletion
Bash(truncate *)         — SQL table truncation
Bash(dd *)               — disk/device write (can overwrite drives)
Bash(mkfs*)              — format filesystem
```

## 5. Hooks

### 5.1 Stop Hook — Task Completion

Fires on the session Stop event.

```json
{
  "type": "command",
  "command": "osascript -e 'display notification \"Claude has finished.\" with title \"Claude Code\" sound name \"Glass\"'"
}
```
Plays the macOS **Glass** sound.

---

### 5.2 Notification Hook — Attention Required

Fires when Claude needs input mid-task.

```json
{
  "type": "command",
  "command": "osascript -e 'display notification \"Claude needs your attention.\" with title \"Claude Code\" sound name \"Ping\"'"
}
```
Plays the macOS **Ping** sound.

---

### 5.3 StopFailure Hook — Error Notification

Fires when Claude stops on an unrecoverable error.

```json
{
  "type": "command",
  "command": "osascript -e 'display notification \"Claude stopped with an error.\" with title \"Claude Code\" sound name \"Basso\"'"
}
```
Plays the macOS **Basso** sound.

---

### 5.4 PreToolUse Hook 1 — lean-ctx Bash Rewrite

Matcher: `Bash|bash`

```json
{
  "type": "command",
  "command": "lean-ctx hook rewrite"
}
```
Intercepts any Bash call and rewrites it to a token-efficient MCP equivalent where one exists (e.g., `grep` → `ctx_search`).

---

### 5.5 PreToolUse Hook 2 — lean-ctx Read Redirect

Matcher: `Read|read|ReadFile|read_file|View|view|Grep|grep|Search|search|ListFiles|list_files|ListDirectory|list_directory`

```json
{
  "type": "command",
  "command": "lean-ctx hook redirect"
}
```
Redirects native file-read and search calls to `ctx_read`/`ctx_search`. Enforces the lean-ctx layer at the harness level, not just in the prompt.

## 9. `settings.local.json` — Local Overrides

Permissions and MCP config specific to the local machine — not committed to any project repo.

### Extra Allowed Permissions

```json
"permissions": {
  "allow": [
    "mcp__next-devtools__browser_eval",
    "Bash(gh auth:*)",
    "Bash(pip3 show:*)",
    "Bash(pip3 install:*)",
    "mcp__github__create_repository",
    "Bash(git clone:*)",
    "Bash(git -C ~/my-project remote -v)",
    "Bash(.venv/bin/pip install:*)",
    "Bash(.venv/bin/jupyter notebook:*)"
  ]
}
```

| Permission | Purpose |
|-----------|---------|
| `mcp__next-devtools__browser_eval` | Allows Next.js DevTools browser evaluation — auto-approved |
| `Bash(gh auth:*)` | GitHub CLI auth commands |
| `Bash(pip3 show:*)` | Show installed pip packages |
| `Bash(pip3 install:*)` | Install pip packages |
| `mcp__github__create_repository` | Create GitHub repositories via MCP |
| `Bash(git clone:*)` | Clone git repositories |
| `Bash(git -C ~/my-project remote -v)` | Check remotes in a specific project directory |
| `Bash(.venv/bin/pip install:*)` | Install into local virtualenv |
| `Bash(.venv/bin/jupyter notebook:*)` | Launch Jupyter notebooks from local venv |

### Additional Flags

```json
"enableAllProjectMcpServers": true
```
Enables all MCP servers defined in project-level `.claude/mcp.json` files without per-server approval.

### Enabled MCP JSON Servers

```json
"enabledMcpjsonServers": [
  "memory",
  "notion",
  "figma",
  "miro"
]
```
These four servers from the system-level `mcp.json` are activated for local sessions.

## 10. `mcp.json` — Custom MCP Server Definitions

One custom MCP server is defined:

### `memory-system`

```json
"memory-system": {
  "command": "path/to/memory-system/.venv/bin/python",
  "args": ["path/to/memory-system/mcp/memory_server.py"],
  "cwd": "path/to/memory-system"
}
```

| Field | Value |
|-------|-------|
| Runtime | Python via local virtualenv inside your `memory-system` directory |
| Entry point | `memory-system/mcp/memory_server.py` |
| Working directory | your `memory-system` directory |
| Purpose | Exposes `search_pinecone`, `search_wiki`, `update_wiki_page`, `ingest_file`, `ingest_url`, `get_log`, `run_health_check` |

Locally-hosted Python server backed by Pinecone (vector search) and a wiki system.

## Summary

| Area | Key choices |
|------|-------------|
| Permission model | `bypassPermissions` — Claude acts freely; destructive ops (rm, force push, DROP TABLE, dd) require confirmation |
| Agent model | Sonnet for subagents via env var; Opus reserved for `advisor()` calls |
| Auto-memory | Disabled — manual memory via claude-mem, knowledge graph, Obsidian vault |
| Hooks | Glass/Ping/Basso sounds for Stop/Notification/StopFailure; 2 PreToolUse hooks enforcing lean-ctx at the harness level |
| Plugins | A handful enabled across multiple marketplaces; context7 plugin disabled (MCP used directly) |
| Custom MCPs | 1 local Python memory-system server (Pinecone + wiki backend) |
| Status line | Custom shell script at `~/.claude/statusline-command.sh` |
