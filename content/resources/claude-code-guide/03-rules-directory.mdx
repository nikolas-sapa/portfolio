> When `CLAUDE.md` gets long, split it. The `rules/` directory holds small domain-specific files — database rules, git rules, UI rules — that you `@import` where needed. Main file stays scannable; context loads only when it matters.

## 2. `design-dna.md` — Frontend Design Rules

Overrides defaults from any design skill, including `frontend-design`.

### Aesthetic
Clean, polished, modern SaaS. Apple-level attention to detail. Liquid glass / glassmorphism where appropriate. Every pixel intentional.

### Bans
- No emojis — use icons (Lucide, Heroicons, or project's icon set)
- No gradient backgrounds (especially blue/purple washes)
- No generic "vibecoded" aesthetic (floating cards on gradient backgrounds)
- No Inter/Roboto/Arial as primary fonts

### Animations
- Right animation for the context — not blanket fade-in
- Physics-based easing (spring, ease-out-expo) over linear/ease
- Staggered reveals, meaningful hover states, smooth layout transitions
- One well-crafted transition beats ten generic ones

### Visual
- Generous whitespace, clear hierarchy, strong typography
- Subtle depth (light shadows, blur, translucency) not heavy 3D
- Restrained color: neutral dominant + 1-2 accents
- Micro-details: consistent border-radius, icon sizing, spacing rhythm

### Self-Improving
On design feedback, update `~/.claude/projects/-Users-<you>/memory/design_preferences.md`. Check it at session start for accumulated preferences.

## 4. `obsidian.md` — Obsidian Vault Rules

**Vault path:** `~/MyVault/`

Use lean-ctx tools directly on the vault — not `mcp__obsidian__`.

### Tool Routing

| Task | Tool |
|---|---|
| Search vault by keyword | `ctx_search(pattern, "~/MyVault")` |
| Read a specific note | `ctx_read("~/MyVault/path/to/note.md")` |
| Map vault structure | `ctx_tree("~/MyVault", depth)` |
| List area or project files | `ctx_tree("~/MyVault/01-Areas", 2)` |

lean-ctx reads the same markdown files faster and with compression.

### Context Loading Order

Before starting any project work:

1. Check if the project's `CLAUDE.md` has `@path` imports — if so, context is already loaded.
2. Otherwise, `ctx_search` for the project name to find relevant notes.
3. Check `01-Areas/_context/` for canonical business docs.
4. Read `02-Projects/[project]/index.md` for current status.

**Never ask the user for something that may already be in the vault — search first.**

### When to Search the Vault

- Starting any project work → read `02-Projects/[project]/index.md`
- Writing any content → load `01-Areas/_context/brand-voice.md`
- Building for a product → load `01-Areas/_context/icp.md`
- User references a meeting, decision, or past note → search vault before asking
- Anything about brand, audience, positioning → check `_context/` first

### `_context/` — Canonical Business Docs

`01-Areas/_context/` is the single source of truth for all business context. Load it before content, copy, or marketing work.

| File | Contents |
|---|---|
| `brand-voice.md` | Tone, personality, writing style, words to use/avoid |
| `icp.md` | Ideal customer profile — who we're building for |
| `positioning.md` | Market position, competitors, differentiators |
| `products.md` | Product summaries, pricing, key features |

### @-Import Pattern for Project CLAUDE.md Files

Add to any project's `CLAUDE.md` to auto-load context at session start:

```markdown
@~/MyVault/01-Areas/_context/brand-voice.md
@~/MyVault/01-Areas/_context/icp.md
@~/MyVault/02-Projects/[project]/index.md
```

### Vault Structure Reference

```

│   ├── _context/          ← canonical business docs (brand voice, ICP, positioning)
│   ├── marketing-growth.md
```

## 5. `ollama-delegate.md` — Ollama Delegation

Delegate bulk/repetitive code generation to Ollama automatically. No user prompt needed.

### When to Delegate
- Boilerplate (components, CRUD, type definitions)
- Unit tests / test stubs
- Simple utility functions
- Documentation / comments

### When NOT to Delegate
- Architecture, complex debugging, multi-file refactors
- Anything requiring skills, MCPs, or deep codebase understanding
- Security-sensitive code

### Delegation Script (Groq → GCP → Local priority)

```bash
# Priority: Groq API (free, fast) → GCP Ollama → Local Ollama
source ~/.config/groq/.env 2>/dev/null

# 1. Try Groq first (free tier: 1K req/day, best models)
if [ -n "$GROQ_API_KEY" ]; then
  RESULT=$(curl -s https://api.groq.com/openai/v1/chat/completions \
    -H "Authorization: Bearer $GROQ_API_KEY" \
    -H "Content-Type: application/json" \
    -d "{
      \"model\": \"openai/gpt-oss-120b\",
      \"messages\": [{\"role\": \"user\", \"content\": $(echo "$PROMPT" | python3 -c 'import sys,json; print(json.dumps(sys.stdin.read()))')}],
      \"temperature\": 0.3,
      \"max_tokens\": 4096
    }" 2>/dev/null | python3 -c "import sys,json; r=json.load(sys.stdin); print(r['choices'][0]['message']['content'])" 2>/dev/null)
  if [ -n "$RESULT" ]; then echo "$RESULT"; exit 0; fi
fi

# 2. Try GCP Ollama
GCP_IP=$(gcp-llm ip 2>/dev/null)
if curl -sf "http://${GCP_IP}:11434/api/tags" &>/dev/null; then
  ENDPOINT="http://${GCP_IP}:11434"
# 3. Fall back to local Ollama
elif curl -sf "http://localhost:11434/api/tags" &>/dev/null; then
  ENDPOINT="http://localhost:11434"
fi

if [ -n "$ENDPOINT" ]; then
  curl -s ${ENDPOINT}/api/chat -d '{
    "model":"<best-available>",
    "messages":[{"role":"user","content":"<specific prompt>"}],
    "stream":false,"think":false,
    "options":{"temperature":0.3,"num_predict":4096}
  }' | python3 -c "import sys,json; print(json.load(sys.stdin).get('message',{}).get('content','ERROR'))"
fi
```

### Model Priority by Tier

| Tier | Models (priority order) | Notes |
|---|---|---|
| **Groq (free, primary)** | `openai/gpt-oss-120b` > `moonshotai/kimi-k2-instruct` > `qwen/qwen3-32b` | Best coding / best agentic / fallback. Free tier: 1K req/day, ~10K tok/min |
| **GCP Ollama (fallback)** | `qwen3.5:27b` > `gemma4:31b` | Spot instance, ~$0.28/hr |
| **Local Ollama (last resort)** | `qwen3:8b` > `qwen3:4b` | |

### Management

- **Groq API key:** `~/.config/groq/.env` (never commit)
- **GCP:** `gcp-llm start` / `gcp-llm stop` / `gcp-llm status` ($300 credits)
- **Auto-start:** When Groq is rate-limited and GCP is down, run `gcp-llm start` without asking.
