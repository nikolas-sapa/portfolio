## Claude Code Settings — `env` Block

File: `~/.claude/settings.json`

```json
"env": {
  "CLAUDE_CODE_DISABLE_AUTO_MEMORY": "1",
  "CLAUDE_CODE_SUBAGENT_MODEL": "sonnet"
}
```

These are injected into every Claude Code process:
- `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1` — disables automatic memory storage (managed manually via memory-system)
- `CLAUDE_CODE_SUBAGENT_MODEL=sonnet` — sets the default model for all spawned subagents

No API keys are stored in the settings `env` block. Shell-level exports for Claude/Anthropic/Groq/etc. are not present in `~/.zshrc`, `~/.zshenv`, or `~/.zprofile` — credentials live in dedicated config files instead.

| Field | Value |
|---|---|
| Credential location | Claude Code CLI manages session auth internally (`~/.claude/` managed files) |

| Field | Value |
|---|---|
| Key name | `GROQ_API_KEY` |

## Key Rotation Checklist

When rotating any credential, update these locations:

1. **Groq** → `~/.config/groq/.env`
2. **Anthropic API** → `~/memory-system/.env`
3. **Pinecone** → `~/memory-system/.env`
4. **Telegram Bot** → `~/.claude/channels/telegram/.env`
5. **GitHub** → `gh auth refresh` (keyring auto-updates)
6. **GCP** → `gcloud auth login` (credentials.db auto-updates)
7. **Vercel CLI** → `vercel login` (access_tokens.db auto-updates)
8. **Stripe** → `stripe login` (config.toml auto-updates)
9. **OAuth MCPs** (Figma/Slack/Notion/Google) → Re-run authenticate tools in Claude Code

> After rotating project-level `.env` files, redeploy or restart any running services.
