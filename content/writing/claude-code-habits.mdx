## The shift

After six months of daily Claude Code use, the patterns that stuck weren't the clever ones — they were the boring, repeatable ones.

## Habits

1. **Plan first.** Use `/plan` before any task over 30 minutes.
2. **Wrap up.** Never leave a session without `/wrap-up` — your future self will thank you.
3. **Name sessions.** Snake-case session titles make memory search actually work.
4. **Skills, not memory.** If you want a repeatable behavior, encode it as a skill. Claude's memory is unreliable; a skill file is not.

## What doesn't work

Vague instructions ("be careful", "always test"). Claude needs specific, actionable instructions. If it matters, write it in CLAUDE.md or a skill.
